- [x] 1. 定位前端/后端保存 config.yml 的实现并复现注释丢失
**关键解决方法**：根因：后端用 js-yaml 的 load/dump 写回 config.yml，会把注释全部丢掉；且 ConfigManager 启动时 validateAndMergeConfig() 还会自动 saveConfig() 覆盖一次。
修复：ConfigManager 改为启动时仅内存合并默认值不落盘；写配置改为对 YAML 文本做按路径的最小化补丁（标量/数组），尽量保留原注释与格式；保存/重置优先使用带注释的 config.example.yml。/api/settings 改为一次性调用 applyPartialConfig。
**不能犯的错误**：避免再次用 yaml.dump 直接重写整个 config.yml（会必然丢注释）；也避免启动时无差别写回配置文件。
