#### 问题分析：

根据用户反馈，CTI工具存在以下问题：

1. **缺少特殊操作字段**：需要添加一个专门字段用于执行特殊操作如Ctrl+C（使用\u0003）
2. **缺少输入验证**：当接收到"Ctrl+C"字符串时，应返回警告并指导用户使用特殊字段
3. **输出内容不完整**：当前只返回"Input sent to terminal successfully"，需要返回命令执行后的实际输出
4. **默认模式问题**：当没有指定mode参数时，应该有合理的默认行为

- 终端管理器：`src/terminal-manager.ts` (第943-1223行的interactWithTerminal方法)

#### 解决方案：

1. 在CTI工具的schema中添加新的`specialOperation`字段
2. 添加输入验证逻辑，检测"Ctrl+C"字符串并返回警告
3. 修改输出逻辑，确保返回命令执行的实际结果
4. 优化默认读取模式，确保在没有mode参数时也能正确返回输出

#### 测试结果：

✅ **特殊操作字段测试成功**：
- 使用 `specialOperation: "ctrl_c"` 成功发送了 Ctrl+C 信号（显示为 ^C）
- 支持三种特殊操作：ctrl_c (\u0003), ctrl_z (\u001a), ctrl_d (\u0004)

✅ **输入验证测试成功**：
- 在input字段输入"Ctrl+C"时，正确返回警告信息
- 提供了正确的使用示例和指导

✅ **默认输出模式测试成功**：
- 不指定mode参数时，默认使用smart模式
- 自动等待1秒并返回完整的命令输出
- 不再只返回"Input sent to terminal successfully"

❌ **发现新问题：输出包含所有历史内容**：
- 在同一终端中执行新命令时，返回了从开始到现在的所有历史输出
- 应该只返回当前命令的输出，而不是整个终端的历史记录

✅ **部分修复成功**：
- 第一次命令执行时正确只返回当前命令输出
- 但后续命令仍包含之前的内容，说明光标追踪需要进一步优化

#### 关键解决方法：

1. **特殊操作实现**：在schema中添加`specialOperation`枚举字段，在处理逻辑中转换为对应的ASCII控制字符
2. **输入验证**：检测input字段中的"Ctrl+C"字符串，返回详细的错误信息和正确使用方法
3. **默认行为优化**：将`waitForOutput`默认设为true，等待时间默认1秒，确保能看到命令结果
4. **智能读取模式**：默认使用'smart'模式，根据内容长度自动选择最佳读取方式
